[个人时光收集器]
（说明：这是为解决我个人“数字记忆混乱”的工具，未来完善目标将贴合我的个人需求）
##项目起源：一个真实的困扰
我的手机和电脑里散落着好几百张照片，它们其中是有着非凡意义的，我需要一个能贴合我的时间线或其他分类要求以及能自定义功能的工具，于是我开始逐步尝试用python创造这样的工具。
##我的解决思路：可塑，透明，私人化
**可塑**：项目代码结构像乐高，我能轻松换时间线，添加新功能（项目持续更新中）
**透明**：所有运行流程都清晰可见，可随时更改
**私人化**：可根据自己的要求创作
##它将如何改变我的
我将不断完善该工具功能，在提升自身编程技能的同时，为自己和家人打造专属的时间路线图



###版本迭代与思考###
**v1.0（初始化化版本，我的第一个编程程序）**
-基础功能：通过文件名提取日期，实现2025到2026按月份分类
-局限性：使用麻烦，没有解决繁琐重复的过程，且时间限制太明显，分类逻辑还补贴我对时间线的要求
-价值：即使第一版本限制多，但它为我python学习指明方向，催生v2.0的升级

**v2.0（当前版本）**
1、改进亮点
-从依赖文件名升级为优先读取EXIF元数据，提升日期准确度
-设计双重保障，先提取EXIF再解析文件名（这点是因为我的拍摄设备上传照片到电脑会自动以日期命名）
-算法优化，现在的分类逻辑是比较贴合我个人人生轨迹的
-代码优化，将os模块大量换成pathlib模块，提高可读性和简洁性
2、**学习收获**
-学习了ExifTags模块后，对字典印象更加深刻，彻底理解键和值之间的特殊关联
-强化了代码调试的能力，学会先根据输出找可能的问题，若无报错也无反馈就只能是逐步缩小范围确定问题所在（但我了解到一种模块能高效处理调试过程-logging）
-纠正了我刚接触python的一些错误概念，如循环语句中只要在循环缩进里不管语句有没有循环变量都要循环执行相应次数等概念
3、遇到的问题
-第二版程序由四部分组成，各个部分如何相互连接嵌套是一个问题，第一部分是路径处理我用列表存储处理后的路径传递给下一部分，第二部分通过这些路径提取对应日期，但第三部分分类该怎么把提取的日期与它的照片对应起来呢，最后想到很贴合该场景的字典，在第二部分以字典形式存储照片路径和对应日期很符合字典的功能
-如何将乱七八糟的日期字符串格式转化为日期对象，我先用replace方法尽可能将格式统一，再转化为日期对象
